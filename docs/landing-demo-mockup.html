Layout Options

Option A: Everything on one page

All 4 sections stacked vertically on the landing page. Pros: everything visible, impressive feature list. Cons: long page, may overwhelm new visitors.

Option B: Main demo + linked sub-pages

Section 1 (main capture) stays on landing page. Sections 2-4 become linked demo pages:

  • /demo/clean — Clean mode comparison
  • /demo/annotate — Screenshot annotation playground
  • /demo/diff — Visual diff tool

Landing page shows cards linking to each demo. Pros: clean landing page, each demo gets full attention. Cons: more clicks to explore.

Option C: Tabbed demo sections

One demo area with a top-level tab selector: Capture | Compare | Annotate | Diff. Each tab shows its own demo UI. Pros: contained, no page navigation. Cons: complex UI, only one demo visible at a time.

Features Described (Not Demoed)

These are listed in feature cards or the API docs section, not as interactive demos:

PDF/A Archival

ISO-standard archival PDFs with embedded fonts. Add pdfa=true to any PDF request.

Timestamped Archive

SHA-256 hashed, RFC 3161 timestamped capture packages. WARC + PDF/A + metadata in one ZIP.

Stealth Mode

Bypass bot detection on Cloudflare-protected sites. Add stealth=true.

Auto-Optimize

Smart format, viewport, and scale selection based on page content analysis.

Structured Extraction

Define a JSON schema, get typed data back. JSON-LD first, LLM fallback for missing fields.

MCP Server

Give any AI agent web access. npx pageyoink-mcp
